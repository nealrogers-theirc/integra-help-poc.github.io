Business Process Information

Key Messages

  1. The Dynad Journal is used for populating and uploading mass journal entries to Integra.
    1. Examples of mass upload journals include payroll allocation, subaward/expenses, re-classification of entries, and shared program cost allocation entries.
    2. Selecting the correct journal name in the Dynad Journal is vital to ensure data is processed correctly in Integra.
      1. Behind each journal name there is systematic coding, i.e., posting profiles, that updates the Integra system in the correct way.
      2. Refer to the appendix at the end of this guide for a comprehensive list of journal names and usage.
    3. The recommendation is that if you have less than ten (10) lines, it may be easier to create the journal entries directly in Integra.
  2. For most business tasks, transactions are posted to sub-ledgers (vs. the General Ledger).
    1. However, there are select transactions that get posted directly to the General Ledger.
  3. Period statuses are used to control access for postings to ledger and sub-ledgers.
    1. For example, vendor (AP) and customer (AR) sub-ledgers should have all invoice/payment journals posted by the sixth working day of month-end closing. To prevent users from posting to AR/AP sub-ledgers after the sixth working day, the sub-ledgers can be “closed” for posting by changing the module status to On hold.

Month-end Close Tasks (High level)

  1. Close/open fiscal posting periods
    • This process is done at specified times in the month by designated Finance individuals.
  2. Ensure all ‘daily’ transactions have been posted in correct fiscal period.
  3. Post intercompany transactions.
  4. Post month-end journals.
    • This process refers to journal entries that are processed at month-end only
  5. Calculate ICR and Prepare Fee Journal
  6. Perform Reconciliation and Analysis
    • This process includes the use of the following standard delivered Integra transactions/functionality to identify and remedy discrepancies
      • Trial Balance
      • Voucher Transactions
      • Accounting Source Explorer
      • Tracing Original Entries
  7. Generate Reports
    • Numerous reports are available for analysis in Integra and Power BI.

Opening and Closing of Ledger Posting Periods

Open or Close a Fiscal Year Period

Purpose

Use this procedure to open or close a fiscal calendar period.

Once you close the current period, the new period should immediately be opened to ensure users can transact from the first day of the next period.

A custom “Period close” security group (PrdCls) has been defined to allow designated Finance users the ability to process periodic transactions after the period has been closed to the general user population.

The following period “statuses” are used to open and close fiscal calendar periods.

  • Open: Open to all eligible users for transaction posting. 
  • On hold: Transactions cannot be posted in the period. However, select users in Finance can post transactions for month-end reconciliation purposes.
  • Permanently closed: This status is currently not being used; once you permanently close a period, it can never be reopened again.

Steps

  1. From the Integra dashboard, go to General ledger > Calendars > Ledger calendars.  
  2. In the left-hand panel of screen, click on the period to change.
  3. Under the Legal entity column, select the legal entity to update. 
Note: After selecting the legal entity, access levels to specific modules can be defined.
  1. In the top left of page, click "Edit".
  2. Select Update Period status. 
  3. Click Edit at the top right-hand side to allow you to change the period status. 
  4. In the Period status field drop down, select On hold/Open for the period that is being closed or opened.   
  5. Under the Ledger column, select PrdCls.
  6. The Ledger column refers to the General Ledger
  7. PrdCls is a custom security group created for specific Finance individuals so they can post transaction after the Period status has been updated to On hold.
  8. If opening or closing a sub-ledger, click Update module access.
  9. For module access update, go to the module that you want to update. 
  10. Select from the drop down, whether you want to restrict a module or user group. If you select “None”, it means that no posting can happen to that sub ledger.  Also, if you select any access level, it overrides the period status.  
  11. All necessary adjusting entries and re-classifications will be done at this point. 

‎ 

 

  1. Click Save. 

Result

You have opened or closed a fiscal posting period.

Checking for Unposted Journals

Check for Unposted Journals for Current Ledger Period

Purpose

Use this procedure to display unposted journals to review and action as required. E.g., post applicable journals.

Steps

  1. From the Integra dashboard, go to General ledger > Journal entries > General journals.

  1. Under the Show heading, the default entry is Not posted. These are the entries to look at to determine what should have been posted in the month being closed.
  2. This entry can be changed to Posted or All by clicking the drop-down arrow.
  3. To determine the date a journal was entered, customize the table grid by adding the column Created date and time.
  4. Right click on any column header and select Add columns.
  5. In the filter box, enter text “Created”.
  6. Select Created date and time from the selection list and Insert it into the table. The column will be added at the end of the current columns.
  7. Review the list of unposted journals in the current period and action as required, e.g., post applicable journals.

Result

You have identified unposted journals for the month-end close period.

Revenue Recognition (HQ Finance Only)

Revenue Recognition is run by HQ. RAI offices are responsible for calculating ICR (Indirect Cost Recovery), preparing the fee journal and submitting the journal to HQ for uploading and posting.

ICR is calculated and prepared in a Fee Template outside of Integra. This template can be downloaded from the following location.

There are two methods available for uploading ICR entries:

  1. Performing a mass upload of entries using the Data Connector Tool (recommended to use when there are more than 10 entries.
  2. Typing ICR entries directly into Integra (option to use when there are less than 10 entries)

Reconciliation Reports

Reconciliation is composed of three key steps.

  1. Running the reconciliation reports to identify discrepancies.
  2. Using Integra’s analysis tools to find out the cause of the discrepancy.
  3. Process journal entries to fix discrepancies.

This topic identifies the reconciliation reports available in Integra, PowerBI or Excel.

Reconciliation Reports Reference Table

Purpose

Use this procedure to run the various reconciliation reports to identify discrepancies.

A common reason differences occur is because a transaction was posted to the General Ledger instead of the sub-ledger. Complete the To-From date fields and make sure you select the accounting currency in the Currency field. Run the report and look at the difference column. It should always be zero. If it is not zero, investigate and post the necessary adjustment entries.

Run Trial Balance Report

Purpose

The Trial Balance is a list page that shows all the balances of an account and/or financial dimensions for a given period. When the trial balance is first opened it refreshes with the balances for the dates and properties that are set in the parameters. Properties that can be changed in the parameters include dates, posting layer, how opening balances appear, and what closing transaction types to show. 

When you change the parameters, the balances are refreshed. You can also pick what dimension set you want to view balances for and whether each of the dimensions show in separate columns. You can also drill down on the balances to view the transactions that make up the balance. 

Steps 

  1. Go to General ledger > Inquiries and reports > Trial balance. 

  

  1. In the From date field, enter the beginning date for the monthly balance. 
  2. In the To date field, enter the ending date for the monthly balance a date. 
  3. Click Calculate balances. 

 

  1. Once the trial balance is completed, you can double-click on the balances to view the transactions that make up the balance. 
  2. Close the transaction by selecting  at the top right of the screen when the review is completed. 

 

 

Result

The Trial Balance has been generated.

  • Monthly Due Dates

Task

Complete Transaction Postings in Sub-ledger Modules

  • Expense Management - process and post all approved travel advances and expense reports (including credit card liquidations and TER)
  • Accounts Payable - Process all vendor invoices and payments.
  • Accounts Receivable - Book all cash receipts and deposits.
  • Cash and Bank Management – Post direct charges, journalize unposted bank transactions as per prior month complete bank reconciliation.
  • Fixed Assets – Generate depreciation proposal
  • Inventory Management – All pick, pack and post transactions (except for "in-kind" Contribution)

Perform Intercompany Accounting

  • Post intercompany entries to A/c 29400 by the 8th of each month for RAI offices still using SUN
  • Post intercompany entries to HQ – 9IRC
  • Move intercompany entries in current period from General Ledger to relevant projects
  • Prepare and Post Monthly General JournalsPost prepaid rent, prepaid insurance, deferred charges, etc.

  • Post depreciation allocation.
  • Post In-kind contribution
  • Post inventory adjustments (if required)
  • Post any required project adjustment entries

Post Allocation journals

  • Post payroll allocation entries
  • Post shared program cost allocation entries

Verify/Check Completion of Tasks

  • All standard journals are posted
  • All unnecessary journals are deleted
  • All unposted journals with a current period date are investigated, amended and/or posted.

Calculate ICR Entries, Post journal and Submit to HQ for Upload

  • Once all of the previous tasks above are completed, calculate ICR entries, and populate the journal
  • Submit journal to HQ Finance for uploading

Complete Sub Ledger to Ledger Reconciliations

  • Run Customer to Ledger reconciliation report.
  • Run Vendor to Ledger reconciliation report.
  • Run Project to Ledger reconciliation report.
  • Run Bank to Ledger reconciliation report.
  • Post all adjustment entries to ensure balances in both sub ledger and ledger are equal

Complete All Other Account Reconciliations

  • Run automated or manual bank reconciliation to reconcile Bank accounts.
  • Perform balance sheet reconciliation and post all adjustment entries to reconcile.
  • Calculate and reconcile Benefit % allocations for Personnel – Salary expenses

Generate Reports once Month-end Close is Complete

  • Generate final Trial Balance
  • Generate Revenue and Expense Report
  • Generate all Project BvA Reports
  • Generate period-end financial statements by using Financial reporter
  • FM01 Report equivalent
