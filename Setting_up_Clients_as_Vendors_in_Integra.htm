Setting up Clients as Vendors in Integra

In Integra, the details of all persons and organizations with whom the IRC interacts with are stored in the global address book. 

The BENE Vendor group relates to any person who is targeted for assistance, who directly benefits from or comes in contact with an IRC-implemented or financially-supported IRC program.  

Most clients are automatically created in Integra through the ETO (Efforts to Outcome) interface; however, non-ETO clients who do not need heightened confidentiality (i.e., their name being saved in Integra is acceptable) can be manually entered and approved by the local requestor. Examples of a requestor include but are not limited to the following roles: 

  • Caseworker 
  • Casework Supervisor 
  • Resettlement Manager 
  • Anti-Trafficking Manager 
  • Finance Manager 

 If the office has a program(s) where clients require heightened confidentiality, Finance will create a generic vendor record in the Vendor module. This is done during the project set-up phase. Example of a such a vendor includes anti-trafficking clients. Due to the sensitivity of this vendor type and to ensure the record is secured to the legal entity, only those with permission can view the record. 

The D9 Financial Dimension

  • One client can have multiple D9 financial dimensions if enrolled in multiple programs.
  • When processing client payment requests, it is critical that the correct D9 is selected, as this is how Integra and ETO track what is spent on each collection and individual client.
  • For client payments where the vendor selected is not the principal applicant, the D9 dimension value is the collection/case number. This number is linked to the principal applicant’s name.
  • For anti-trafficking clients, before requesting the first payment, the client ID (D9) must be created in Integra. When requesting the payment, select the generic vendor record as the vendor.

Process Diagram

Process Overview

Caseworker enters client payment request in Integra.

Caseworker submits payment request to workflow for approval.

Once approved by supervisor and Finance, caseworker prints hard copy of payment request.

Finance posts the payment request.

Finance processes payment in Integra.

The client’s date of birth is required for payment. Refer to the procedure in this document titled, Display List of Direct Assistance Cases for instructions on finding this information.

Caseworker gives prepaid card (or check) to client (or vendor, when applicable) and obtains client signature on hard copy printout of payment request.

Caseworker attaches electronic copy of signed document to payment request journal in Integra.

Caseworker files hardcopy in client’s folder.

Recommended Follow-on Steps

  • Caseworker verifies that the remaining budget has been updated on the Client Payment Dashboard.
  • Caseworker ensures that payment transferred to ETO.

Process Payment Request for a Client

Purpose

Use this procedure to enter a client payment request journal in Integra.

Each payment request journal requires two lines: a vendor line and a project line.

Note: If you are including multiple vendor lines in one journal (all using the same Client ID), each vendor line must have a corresponding project line.

There are different financial dimension requirements for each line in the payment request journal:

Vendor line requires D1, D4, and D5

Project line requires D1, D2, D3, D4, D5, D6 and D9

Attention: Ensure you attach the client-signed document at the journal batch level, not on the individual journal voucher lines.  

Prerequisites

Client has a BENE vendor record set up in the Integra system. If a BENE vendor record does not exist for the client, each RAI office has one or more individuals who can create the vendor record.

All financial dimensions are known
‎

Upload Signed Copy of Invoice to Journal Invoice Record

Purpose

Use this procedure to attach the client-signed invoice to the posted invoice journal in Integra.

IMPORTANT: The invoice must be attached at the journal batch level, not the journal voucher line level.

Prerequisites

  • Finance has posted invoice journal and issued payment to client.
  • Client has signed the hard copy of the invoice journal.

‎

Steps

  1. Access the Client Payment Dashboard page and select the tile Payment requests (Invoice journal).

  1. Under Show, click the drop-down arrow and select Posted.
Note: This field always defaults to Not posted so this must be changed to Posted each time a signed invoice needs to be uploaded.
Tip: Select the checkbox next to Show user-created only to narrow down the list to only display journals you have created.
  1. Select the checkbox to the left of the relevant journal batch number.

  1. To upload/attach a file, click the paperclip in the top right corner tool bar.

  1. On the menu bar at top, click New and select File from the drop-down list.

  1. Click Browse and locate and attach the signed document.

A message will appear when the upload is complete. The number on the paperclip icon will also update to reflect the number of attachments on the record.

  1. Close the attachment window to return to the invoice journal page.
Note: To view all attachments at any time, simply click the paperclip icon and the attached documents are displayed in the left-hand pane.

Result

You have uploaded the client-signed invoice to the posted invoice journal in Integra

Requesting Client Payment for Minor Clients (under 18 years)

Minor clients are enrolled in ETO program (typically workforce development, youth or safety and wellness).

The minor client is created via the ETO/Integra interface and appears on the direct assistance case table.

As the client is under 18, no vendor record is created. We do not manually create these records as there is an anticipation that the record will be created automatically when the client turns 18 and we do not want duplicates.

To make a payment directly to the minor child, the generic client record is selected as the vendor record (e.g., Atlanta Client) and the correct client ID is tagged to the transaction using the same steps as any other client payment.

As long as the correct ETO Program is tagged to the Project ID and the correct D9 is selected, this transaction should flow over to ETO, the same as any third-party payment
